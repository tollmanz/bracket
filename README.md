# Bracket

Retro arcade scoreboard for our annual NHL Stanley Cup playoff pick 'em. Every season
we have data for lives in one place: an all-time leaderboard plus a page per year.

It is a static site generated by [Eleventy](https://www.11ty.dev/). The JSON in `data/`
is the source of truth; JavaScript data loaders read it at build time and Eleventy
pre-renders every page to plain HTML. There is no client-side JavaScript.

Each player predicts the winner and game count of every series. Correct winner earns the
round's base points; nailing the game count too adds a bonus. See `data/rules.json`.

## Running

```
nvm use
npm install
npm start      # eleventy --serve, then open the printed localhost URL
npm run build  # one-off build into _site/
npm run verify # recompute scores and check documented totals
```

## Layout

```
data/                 source of truth (facts only, hand-edited)
  index.json          years + status/champion, default year
  rules.json          base points per round, games bonus
  people.json         player ids, display names, aliases
  <year>.json         one file per season
lib/
  score.mjs           pure scoring + all-time aggregation (shared)
  render.mjs           HTML builders (page shell, leaderboard, year scoreboard)
src/
  _data/bracket.js    build-time loader: reads data/ for every template
  index.11ty.js       all-time leaderboard at /
  year.11ty.js        one page per season at /<year>/ (Eleventy pagination)
assets/styles.css     pixel-art styling (passthrough-copied to _site/assets/)
eleventy.config.js    input src/, output _site/
scripts/
  verify-data.mjs     recomputes scores, checks documented totals
  import-build.mjs     one-time importer that produced data/ from sources/
sources/              raw inputs the data was built from (provenance)
_site/                build output (gitignored)
```

## The leaderboard

The index aggregates across every season:
- Championships: first-place finishes in completed pools (ties shared)
- Rounds won: top score in any fully-final round, across all years (ties shared)
- Total points and seasons played

Ranked by championships, then rounds won, then total points.

## Data model

Year files store facts only. Points and standings are computed from `rules.json` in
`lib/score.mjs`, so nothing derived is stored and totals cannot drift.

```jsonc
{
  "year": 2026,
  "status": "in-progress",        // or "complete"
  "champion": "VGK",              // final winner, null while in progress
  "matchupsReconstructed": false, // true for 2018-2022 (opponents recovered via research)
  "source": "...",
  "rounds": [
    {
      "round": 1,
      "name": "Round 1",
      "series": [
        {
          "teams": ["COL", "LAK"],
          "result": { "status": "final", "winner": "COL", "games": 4 },
          "picks": [{ "person": "zack", "team": "COL", "games": 6 }]
        }
      ]
    }
  ]
}
```

Team codes are canonical 3-letter NHL abbreviations (`TBL`, `CGY`, `WSH`). Player emails
in `people.json` are stored as SHA-256 hashes (`emailHash`), not plaintext.

## Adding a season

1. Add `data/<year>.json` following the model above
2. Add the year to `data/index.json`
3. Add any new players to `data/people.json`
4. `npm run verify` then `npm run build` (the year page and leaderboard update automatically)

## Hosting

`_site/` is a plain static site; deploy it to any static host. Pages use root-relative
paths, so for a hosting subpath (e.g. a GitHub Pages project site at `/bracket/`) set
Eleventy's `pathPrefix` and use the `url` filter.

## Data sources

- 2026: the current season's picks
- 2025: the `Projects/picks` markdown pool
- 2018-2022: the original `tollmanz/bracket` repo. Those files recorded only the winning
  team and game count per round, so series opponents were reconstructed from historical
  playoff results and verified against the recorded winners and the known Stanley Cup
  champion. 2022 source data covers only rounds 1 and 2.

`@TTollman` (Tyler Tollman) appears only in 2019-2020, and `@thoronas` is Flynn O'Connor
under an old handle (merged into `flynn`). The remaining early-years handles `@nacin`,
`@brothernacin`, `@fathernacin`, `@cklosowski` are stored under their handle; edit
`data/people.json` to give them a display name.
